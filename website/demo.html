Attack-aware autoscaling

Measure live traffic, classify stress, and defend at the edge — before the catalog pays for abusive load.

This interface mirrors a production-style path: client → API gateway → catalog service. Select a threat profile, run the controlled surge, and compare reactive scaling with edge blocking using the same live metrics, policy engine, and storefront iframe — no mocked route.

Why this is not “just microservices + ML”

Naive autoscaling

Elevated RPS or CPU → add replicas. Malicious traffic still hits the catalog; you spend compute and capacity serving requests that should be refused.

Attack-aware gateway policy

A sliding window drives classification; under attack-like stress the gateway returns HTTP 429 on /products and stops forwarding. Defence clears only through an operator restore — same route as the live shop, not a hidden test endpoint.

EDGE

Threat contained at the gateway. Abusive /products traffic receives 429; the catalog is no longer absorbing this load. Policy-driven defence — not blind horizontal scale.
